CSS 변환,전환,애니메이션

> CSS 변환,전환,애니메이션 이 3개는 작성법이 까다로워 2탄으로 따로 정리 해볼게요. ### transform ``` ``` transform은 변환 함수를 뜻합니다. 보통은 개체의 위치 이동이나 회전 등에 사용됩니다. transform에 들어가는 변환 함수는 이렇게 있습니다:
  • 2차 변환 함수
    • translate함수
      • translate(x,y) : 이동(x축,y축)
      • translateX(x) : 이동(x축)
      • translateY(y) : 이동(y축)
    • scale함수
      • scale(x,y) : 크기(x축,y축)
      • scaleX(x) : 크기(x축)
      • scaleY(y) : 크기(y축)
    • rotate함수
      • rotate(-deg) : 회전(각도)
    • skew함수
      • skewX(x) : 기울임(x축)
      • skewY(y) : 기울임(y축)

  • 3D 변환 함수
    • perspective함수
      • perspective(-px) : 원근법(거리)
    • rotate함수
      • rotateX(x) : 회전(x축)
      • rotateY(y) : 회전(y축)
      • backface-visibility : 뒷면 숨김 여부 함수, visible (default값) 혹은hidden으로 설정 가능.

**주의할 점은 transform의 단축속성 작성법이다. transform : 원근법 이동 크기 회전 기울임; 이라는 것을 잊지말자!** ### transition ``` ``` transition은 요소의 시작과 끝의 전환효과를 지정하는 것이다. 지속되는 시간, 전환 효과의 속도나 대기시간을 지정할 때 사용한다. transition에는 이렇게 있다▼
  • transition-property :
    • all : 모든 속성에 적용
    • 속성이름 : 전환효과를 적용할 속성의 이름

  • transition-duration : 지속시간 설정
    • 0s : 전환 효과 없음
    • -s : -(지속시간)s 설정

  • transition-timing-function : 전환효과의 타이밍함수를 지정하는 것으로 값으론 이렇게 있다.↓
    • ease : 느리게-빠르게-느리게
    • linear : 일정하게
    • ease-in : 느리게-빠르게
    • ease-out : 빠르게-느리게
    • ease-in-out : 느리게-빠르게-느리게

  • transition-delay
    • 0s : 대기시간 없음
    • -s : -(대기시간)s 설정
**주의할 점은 transition의 단축속성 작성법이다. transition** : 속성명 **지속시간** 타이밍함수 대기시간; 이라는 것을 잊지말자!!
### animation ``` ``` animation은 말 그대로 요소에 애니메이션 효과를 적용할 때 사용하는 코드입니다. 먼저 @keyframes로 만들고 싶은 애니메이션 효과를 지정하고, 추가하고 싶은 태그에 animation을 작성하여 사용합니다. 또한, animation을 작성할 때 위의 코드와 같이 몇 개의 요소들을 지정해야 합니다.
  • animation-name : @keyframes로 지정한 애니메이션 이름 작성
  • animation-duration : 애니메이션의 지속시간 설정
  • animation-delay : 애니메이션의 지연시간
  • animation-interation-count : 반복되는 횟수 지정. 숫자, 소수점, infinite로 지정
  • animation-timing-fuction : 애니메이션이 진행되는 형태 설정, 위 transition과 동일한 내용 작성가능
  • animation-direction : 애니메이션의 방향 설정 ,normal과 alternate가 있다.
**주의할 점은 animation의 단축속성 작성법이다. 순서는 animation: example duration timing-function delay iteration-count direction fill-mode;이며 duration과 delay는 안 쓰더라고 꼭 작성 필요!**
